# 2026-04-03

## 工单统计表格分析 - 增强发件人过滤

用户让我学习 `analysis.xlsx` 自动化工单统计表格，发现很多通知类工单可以通过发件人提前筛掉。

### 分析结果

表格包含4个sheet：
- **正确指派** (5条数据): 正确关闭的系统通知
- **错误指派** (~157条数据): 大量错误需要关闭/重新指派的工单
- **正确起草回复** (1条): 缺货回复模板
- **错误起草回复** (6条): 用了错误的回复模板

### 主要问题

1. **Amazon 多语言通知漏关** - 大量德语/意大利语/西班牙语/法语/芬兰语的 Amazon 系统邮件被误指派给各 agent
2. **AliExpress 通知漏关** - 违背发货承诺、订单关闭等速卖通通知没被过滤
3. **Fuuffy 物流追收费用** - 运单差价/派送延误通知
4. **Mailchimp/PayPal/Facebook** 系统通知未完全覆盖
5. **订单咨询误指派** - Shopify 订单相关邮件经常错派给 Jennifer/Gwen 而不是 Lena

### 改进措施

更新了 `triage.js`：
- **Layer 1a (发件人)**: 大幅扩展发件人黑名单，增加 Amazon 全球 TLD、AliExpress seller/operation、Meta、PayPal noreply、TikTok、通用 @noreply/@mailer/@bounce 模式
- **Layer 1b (内容)**: 精简为纯内容层，增加 Amazon 多语言通知关键词、AliExpress 通知、Fuuffy 追收、音乐版权、Facebook/Meta 系统通知
- **Soft spam**: 增加更多模糊判断规则

同时更新了 `SKILL.md` 文档。
